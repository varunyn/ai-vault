# Report Design Reference

Use a quiet, editorial, product-internal visual style: generous whitespace, strong card titles, muted sidebar labels, and subtle borders. Keep the report as a single HTML file with embedded CSS and minimal interaction JavaScript.

Use semantic `aside`, `main`, `section`, `article`, `nav`, and native `details` elements. Keep the sidebar sticky on desktop, responsive on mobile, keyboard-accessible toggles, visible focus states, and status labels that do not rely on color alone.

Structure the page with a title, a date or date range, optional intro text, an anchor sidebar, and independently readable work cards. Each card should include a date/time label, concise title, status badge, short summary, expandable details, and source metadata. Prefer 3–8 concrete detail bullets when source material supports them.

Use `done`, `in-progress`, `blocked`, and `note` status labels. The quality bar is a report that is scannable in one minute, preserves meaningful engineering outcomes, avoids transcript noise, and is polished enough to share internally.
